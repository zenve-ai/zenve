# Zenve Artifact PR System

## Overview

This document defines the full implementation of Artifact PRs in Zenve,
including: - Purpose and design - Agent behavior - PR rules - Auto-merge
flow - Validation - Pipeline transitions - Failure handling

------------------------------------------------------------------------

## Pipeline

    zenve:pm
      ↓
    zenve:arch
      ↓
    zenve:dev
      ↓
    zenve:reviewer
      ↓
    done

------------------------------------------------------------------------

## Problem

PM and Arch generate files (PRD, architecture docs), but if those are
created in PRs that are not merged, the next agent cannot see them.

------------------------------------------------------------------------

## Solution: Artifact PRs

Artifact PRs are automatically created and merged within the same run.

### Behavior

1.  Agent runs in isolated worktree
2.  Generates files
3.  Zenve commits changes
4.  Zenve creates PR
5.  Zenve validates changed files
6.  Zenve auto-merges PR
7.  Zenve syncs repo
8.  Zenve transitions issue

------------------------------------------------------------------------

## Agent Modes

    pm        → artifact_pr
    arch      → artifact_pr
    dev       → code_pr
    reviewer  → no_pr (read_only before, no work tree)

------------------------------------------------------------------------

## PR Rules

### DO NOT close issues

Allowed:

    Refs #123
    Related to #123

Forbidden:

    Closes #123
    Fixes #123
    Resolves #123

Only Dev PR closes issue.

------------------------------------------------------------------------

## PR Title

    [zenve][pm] PRD for #123
    [zenve][arch] Architecture for #123

------------------------------------------------------------------------

## PR Body Example

    Refs #123

    Generated by Zenve PM agent.

    Artifact PR – safe to auto-merge.

    Files:
    - docs/product/issue-123-prd.md

------------------------------------------------------------------------

## Allowed Paths

Example:

    docs/**

------------------------------------------------------------------------

## Validation

All changed files must match allowed paths.

If any file is outside allowed paths:

-   DO NOT merge PR
-   Mark issue as failed

------------------------------------------------------------------------

## Auto-Merge Flow

    agent completes
    → commit
    → push branch
    → create PR
    → validate files
    → merge PR (squash)
    → sync local repo
    → transition issue

------------------------------------------------------------------------

## Sync After Merge

    git fetch origin
    git reset --hard origin/main
    git clean -fd

------------------------------------------------------------------------

## Pipeline Transition Rule

Only transition AFTER merge.

Correct:

    merge PR → transition issue

Wrong:

    transition → PR still open

------------------------------------------------------------------------

## Failure Cases

### Invalid file changes

-   Block merge
-   Comment on issue
-   Label zenve:failed

------------------------------------------------------------------------

### Merge conflict

-   Try rebase
-   If fails → zenve:needs-input

------------------------------------------------------------------------

### GitHub API failure

-   Retry
-   If fails → zenve:failed

------------------------------------------------------------------------

## Summary

Artifact PRs:

-   keep worktree isolation
-   ensure visibility of artifacts
-   avoid blocking pipeline

Flow:

    PM → artifact PR → auto-merge → Arch
    Arch → artifact PR → auto-merge → Dev
    Dev → PR → Reviewer
